1. Networked Applications

May 31, 2022

Connectivity: two computers in different parts of the word can connect to one another and exchange data.

Basic model: two computers, each running a program locally, two programs communicate over network.

most common communication model used: bidirectional, reliable stream of bytes.

  • program A on computer A: write data goes over network
  • program B on computer B: read

Byte Stream Model

  • computer B: waiting for other computer to connect to it (eg. web server)
  • computer A: connect to computer B (eg. mobile phone)
  • A and B setup a connection
  • A write data to connection
  • data travel through network that B can read it
  • similarly, if B write data to the connection, data travel through the network that A can read it
  • either side can close the connection (eg. web browser is done requesting data from web server)

Word Wide Web (HTTP)

HTTP: HyperText Transfer Protocol

A client opens a connection to a server and send commands to it (eg. GET-request a page)

HTTP : document-centric way for programs to communicate

  • type ‘http://www.cmu.edu/' in browser
  • the browser opens connection to the server www.cmu.edu, and sends a GET request for the root page of the site
  • the server receives the request, check validity, and sends the response (associated with numeric code. eg. 200 OK, 400 Bad Request)

because HTTP is document centric, client requests name a file. ASCII

client sends a request by writing to the connection, the server reads the request, processes it, and writes a response to the connection, which the client then reads.

BitTorrent

In BitTorrent a client requests documents from other clients. Unlike the web, where client requests documents from a server.

A single client can request from many others in parallel.

BitTorrent breaks files up into chunks of data called pieces. When a client downloads a complete piece from other client, it then tells other clients it has that piece so they can download it too.

These collections of collaborating clients are called swarms.

BitTorrent uses the exact same mechanism as the word wide web: a reliable, bidirectional byte stream. But in a slightly more complex way.

When a client was to download a file, it first has to find a torrent file. The torrent file describes some information about the data file to download. It also tells BitTorrent about who the tracker is for the torrent.

A tracker is a node that keeps track of what clients are members of the swarm.

To join a torrent, your client

  • contacts the tracker over HTTP, to request a list of other clients.
  • opens connections to some of these clients and start requesting pieces of the file. Those clients, in turn, can request pieces on their own.
  • When a new client joins the swarm, the tracker can tell the client to connect to your client.

Rather than a single connect between the client and the server, you have a dense graph of connections between clients, dynamically exchanging data.

Skype

simple client-server exchange

The caller open a connection to the recipient. If the recipient accept the call, you start exchanging voice, video or chat data.

In Skype case you have two clients, two personal computers requesting data from each other.

The complication comes from NAT, Network Address Translator. (eg. home wireless router)

If you are behind the NAT, you can open connections out to the internet, but other nodes on the Internet can’t asily open connections to you.

Skype using Rendezvous server to work around NAT.

  • when login to Skype, Client B opens connections to a network of control servers (rendezvous server)
  • when Client A calls Client B, it sends a message to the rendezvous server. Since the server has an open connection to client B, it tells B that there’s a call request from A.
  • if Client B accepts the call, then it opens the connection to Client A

reverse connection - reverse the expected direction for initiating the connection.

Personal computers are often behind NATs for security and other reasons.

Both behind NATs? - Relay

both clients open connections to the relay. When client A sends data, relay forwards it to client B through the connection that B opened.

Word Wide Web - client-server model : a client opens a connection to a server and requests documents.

BitTorrent = peer-to-peer model: swarms of clients open connections to each other to exchange pieces of data, creating a dense network of connections.

Skype is the mix of the two: when skype clients can communicate directly, peer-to-peer fashion. otherwise go through rendezvous or relay servers.
