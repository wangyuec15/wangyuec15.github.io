Recursion

Code Optimization

What is Recursion

  • Defining a potentionally infinite set of objects or computations with a finite statement by
    defining the set in terms of a varition

Use cases for recursion

  • Divide-and-conquer algorithms
  • The basis for Dynamic Programming

The different Cases

  • Base case
    • terminates recursion, generates the r4esult which ultimately produces the final answer
    • a missing or bad case results in infinite recursion
      Trivial Case - outputs a result with no recursion
      Recursive Case - outputs with one or more recursions

What If There Is No Base Case

  • deal with this bu using a parameter specifying when to stop
  • number of recirsions
  • threshold on the value
  • or some stopping criteria

Recurrence Relations

  • describes the sequence of values as you iterate an input value by giving

Example: Factorial/Fibonacci

Recursive Data Strasctures

  • linked lists
  • binary trees

Using

  • accessors
  • merging
  • adding
  • deleting
  • mutating

Recursion and Iteration

  • Many algorithms can eliminate the stack when turned into iteration

  • parameter to fixed value

    • the fixed value becomes the base case
    • the parameter doubles as the iteration index

  • fixed value to parameter

    • need a second parameter to direction as other action index

Tail Recursion

recursive call is the very last thing done by the function

  • Instead of making the recursive call, just update the function parameters and jump back to the function’s entry point
    • reduce stack space from O(n) to O(1)

Computational Complexity

  • Draw the call tree
  • Then either
    • determine the height of the tree
    • determine the work done each level of the tree

Code Optimization

  1. Make sure that you are using the proper algorithm
  • optimizing an O(N^3)
  1. find out which parts of your code need to be faster
  • reduce function nesting
  • strength reduction
  • shift instead of multiply/divide by a power of 2
  • Common subexpression Elimination
  • Induction Variables
  • Loop Unrolling
  • Avoiding Branches
  • remove reference
  • Cache Awareness
