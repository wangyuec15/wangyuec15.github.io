Sep 13, 2022

Searching:

  • Unsorted Array
    • linear search O(n)
  • Sorted Array
    • binary search O(log n)
  • Unsorted Linked List
    • O(n)
  • Sorted Linked List
    • O(n)
  • Unsorted Tree
    • O(n)

Tree traversal

  • Enumerating all of the nodes in a binary tree
  • Pre-order, In-order, Post-order

Binary Search Trees

  • Binary trees
    • each node has a value, a left child and a right child
    • the value of left child always < value of node
    • the value of right child always > value of node
  • Simple search
    • compare key against node’s value

BST Search Complexity

  • O(height), which is only O(log n) for a balanced tree
  • Worst case: O(n) (only one child for each nodes)

BST Balancing

  • local rotations

Implicit Binary Trees

  • array
    • a[i]’s children are a[2i+1] and a[2i+2] if a[0] is the root
    • if the tree is complete, there are no unused element in the array
    • or just use sorted array!

General Trees

  • a tree with up to k children per child is called a k-ary tree
  • any k-ary tree can be sorted in a binary tree
    • ‘left’ points as first child, ‘right’ as siblings

Special k-ary trees: Tries

  • also called radix tree or prefix tree
  • like a radix sort, treats the key as a number and operates on one digit at a time
  • nodes in a trie do not store keys
    • the node’s key value is implicit in its position in the trie
  • can be used to replace a hash table
  • lookups are O(keylength), just as in a hashtable

(alphabet)

  • one option: use terminators
  • store flag or special values in node

Common Uses

  • prefix searches
  • auto completion
  • spell checking

Path Searches

  • what if we need to find a path to an item, rather than the item?
  • if we need to generate the actual path rather than just its length, also need to remember the nodes

Graphs

  • Trees are special case of graphs
  • Graphs consist of a set of vertices and a set of edges
  • Two common representations
    • adjacency matrix
      • (N*N) matrix can be weighs 1/0 for present/absent
    • adjacency lists
      • each node stores a list of its neighbors reachable via single edge
      • can also store weight for each neighbour

Graph Search

Depth First Search

  • start at origin vertex
    • if no valid neighbours, failure
    • for each vertex reachable from the current, recursively search using neighbor as the origin
    • stop as soon as a recursive search is successful, and return success
    • if none of the recursive calls successful, failure

reachable Vertices

  • taking every neighbor works for tree and directed acyclic graphs
  • but if there are any loops, need to remember where visited

Depth-first Search: Shortest Path

Breadth-First Search

  • Searches the graph in order by distance from the start node
  • need to maintain a queue of nodes to be visited
  • Start by putting origin vertex on queue
  • while(!queue.empty)
    • pop vertex from queue
    • if target, return success
    • otherwise, add vertices reachable from there to queue
