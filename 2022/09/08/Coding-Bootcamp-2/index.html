Lomuto Partitioning in Action

Hoare Partitioning in Action

  • advance left past elements less than pivot
  • advance right until we see an element less than or equal to the pivot

Complexity: Merge Sort

  • first pass merges N/2 pairs of elements
  • second pass merges N/4 pairs of two-elements runs
  • third pass merges N/8 pairs of four element runs
  • …
  • each pass does O(n) work
  • there are O(log n) passes
  • so the total for the entire sort is O(log n)*O(n) - O(n log n)

Complexity: Quick Sort

  • Average case: we split the array approximately in half
    • so the recursive depth is O(log n)
    • each layer of recursion does a total of O(n) work
    • so the total is O(log n)*O(n) = O(n log n)
  • Worst case
    • O(n^2)

Timsort

  • A hybrid of Merge Sort and Insertion Sort
  • find runs in the data and processes the runs according to their size
  • more complex than Merge Sort, but gains efficiencies on real-world inputs
  • now the standard sorting algorithm in Python

  • same APIs as .sort(), but runs in multiple threads

In Coding Interview

Step 1: Read the Problem statement

  • figure the actual problem to solve from narrative
  • Determine the type of problem
  • string manipulation
  • sorting
  • search
  • optimization(best path/order/selection)

Step 2: Figure out Possible Decompositions

  • Multiple subtasks may be easier to solve individually
  • Multiple subtasks may each need a different algorithm or data structure
  • If one or more in a version of real problem

Step 3: Analyze the constraints

  • Are there limits on algorithm time/space
  • Are there limits on input values
  • Are there limits on input size

    Step 4: Enumerate Candidate Algorithms

  • a set of candidate algorithms, okay to include b

Step 5: Adapt Algorithms

  • try adapt known algorithms via minor modifications
  • try adapt known algorithms by changing interpretation of data

Step 6: Select Best Algorithm For Each Part

  • compare time/space complexities
  • compare coding difficulty
  • take the constraints into account

Step 7: Connect Sub-Problems

Design overall solutions

Step 8: Try out examples

  • Manually run through a small example or two using the chosen solution
  • If don’t get the right answer, recheck choice of algorithms, check connections between algorithms

Step 9: Figure out edges / boundaries

special input: max, min…

Step 10: Write Code

detailed solution hand check

Step 11: Test and Debug

double check calculations, make sure handle boundaries

edge input of string

  • completely empty input
  • switches only
  • multiple consecutive strings
  • empty switches (slash followed by blank)
  • empty negated switches (/-)
  • dash following one or more switch characters (/A-notaswitch)

JavaCommand Using Regular Expressions

  • Switch is matched with “/-?[A-Za-z]*”
  • To extract just the non-switch strings, convert switches to blanks, then merge away extraneous blanks and split on blanks
    • cmdline.replaceAll("/-?[A-Za-z]*","").replaceAll(" +", "").replaceFirst("^+","").replaceFirst("+$"."").split("")
      (two blanks before +)
  • To get the switches, build a regex matcher and apply find() in a loop
    • Matcher m=Pattern.compile("/-?[A-Za-z]*").matcher(cmdline)
    • while (m.find()) ( String sw=m.group().toUpperCase0;...)
