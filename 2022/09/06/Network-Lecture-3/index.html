Build a Local Area Network

Step 1: Choose an addressing scheme (example: Mac addresses)

  • Mac Address - “Media Access Control Address”
    • 48 bits long , written as a sequence of hexadecimal numbers
    • e.g. 34:f3:e4:ae:66:44
    • 2^48 possible
  • Ethernet Packet - AKA “Frame“/“Datagram“ Never “Package”!
    • header + payload
      • Preamble: Always 10101010101 repeating, 56 times - synchronizing !
      • SFD
      • Destination Address - All MAC Address is assigned by manufacturer! (like phone nums)
      • Source MAC Address - who to reply to
      • EtherType - what kind of data is inside packet
    • Payload - data you want to send!
    • FCS - chack if data has been corrupted

Step 2: choosing a routing algorithm

solutions:

  • Hard-Code Everything (bad: hard to update, people make code mistakes, replicating un-efficiently, new-joiners and mobility, poor scalability)
  • Better solution: Plug and Play
    • Self-update network
    • Broadcast: send packet to every link, node copy packet and send
    • No collisions in switch network, each one has sender and receiver
    • Problem: privacy, receive duplicate, redundant, circularity
    • Smart thing: Learning Bridges
      • What happens when receivers replied?
      • Switch maintain the table: “send to which Mac address on which port”
      • When packet received:
        • if not exists, insert source to table
        • if destination Mac address in table: jus the port; else, send out all ports
    • Pros: Plug and play, O(n=node count)
  • Solutions: make your networks in a tree
    • Spanning Tree Protocol
      • No loops in topology
      • All Lan segments
