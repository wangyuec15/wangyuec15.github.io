Git笔记8-GitLab实践

Aug 29, 2021

57: GitLab的价值

开源 可以自己搭建社区环境 进行二次开发
维护人员很多，新功能多

界面布局条理化，易用性好
集成CI的功能，性能快
研发全周期的完整解决方案

58: GitLab的核心功能

管理、项目管理、仓库管理、verify、打包、发布、配置、监控、安全保证
一个产品完成所有的事情
dev ops
数据统计
计划 看板 跟踪管理任务需求
create 代码管理 分支管理
verify 主干质量保证 集成申请的代码review限制，各种检查手段 自动化测试和报告
release pipeline功能，面向多服务器部署
配置功能 环境 权限
监控 代码变更对生产环境的影响

59: GitLab上怎么做项目管理

GitHub：issue、 merge request
GitLab：issue label 看板
service desk 和用户沟通 客户支持
milestones 里程碑管理

60: GitLab上怎么做code review

把分支做成保护 团队成员不能对保护做push
快捷键ctrl+Enter
自然人看代码
自动化review gitlab bpt
指定给谁 milestone label 相关的merge request
左下角settings 保护分支 强制要求merge request

61: GitLab怎样保证集成的质量

gitlab-ci.yaml
在pipeline的阶段看任务阶段组织 并行任务
gitlab runner 测试 agent 跑cicd
可以搭自己的runner或共享runner

62: 怎样把应用部署到AWS上

yaml中 deploy指定分支
settings - 保护分支 - allow to push: no one
commit调整
部署的脚本deploy文件夹
aws的 DNS ping时间
在GitLab test 检查commit
merge request的 target branch就是要合到的branch
settings - general - merge request 可以设置区别于GitHub的功能fast-forward merge
有deploy文件说明已经部署了
