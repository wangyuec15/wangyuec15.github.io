1.课程综述

版本控制系统

原始​：目录拷贝，公共文件容易被覆盖，沟通成本高，效率低
​发展​：集中式vcs，版本管理，分支管理。客户端没有完整历史，必须随时与服务器相连
​分布式vcs​：服务端和客户端都有完整版本

git特点

最优的存储能力，非凡的性能，开源的，容易备份，支持离线操作，容易定制工作流程

10. commit tree和blob的关系

commit 每一次执行git commit都会创建一个这样对象
一个commit对应一棵树 一个快照 存放了当前commit对应的本项目仓库所有文件夹和文件的快照
tree文件 里面包含树（文件夹） 有哈希值 blob blob指具体的文件
blob和文件名没有关系 只有唯一的内容 节约存储空间

commit里的树 那个点看到的项目所包含的内容 叶子节点是文件

12:分离头指针情况下的注意事项

detached head

打开git log

基于commit建立checkout
分离头指针
可以继续开发，产生commit，并且不会影响其他分支
在没有分支的情况下commit并且变更，切换到master，分离头指针的变更可能会丢失
commit (HEAD)
继续checkout到master
提示：要不要为分离头指针的commit 建branch
重要变更一定要和分支绑在一起
