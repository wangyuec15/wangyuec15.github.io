Git笔记2-独自使用Git时的常见场景

Aug 28, 2021

14:怎么删除不需要的分支

-d报错not fully merged 确定删除-D

15:修改最新commit的message

16:修改老旧commit的message

变基rebase
进入交互式界面，在前面加r 保存退出，进入另一个交互界面
可以改变message，再次保存退出
commit的id改变了，但是内容没有变
在log中可以看到改变后的message在原来的提交位置

17:把连续的多个commit整理成一个

在交互式页面中可以看到pick
s, squash 合并到前面的commit里
将要向前合并的commit前缀改为s 保存退出
写一句合并message 保存退出
git log可以看到合并了commit

18:把间隔的几个commit整理成一个

如果要合并最新的，交互式页面没有要加上
将要合并的移动到相邻的位置 前缀为s

加入新消息，保存

19:比较暂存区和HEAD所含文件的差异

变更，git add 文件
status会看到有文件被修改

必须加cached查看暂存区和HEAD之间的差异

20:比较工作区和暂存区的差异

可以加具体的文件名

21:让暂存区恢复成HEAD的一样

工作区的变更比暂存区的更好，暂存区中所有内容都不想保留

unstaged changes
git diff –cached返回是空的，说明暂存区和HEAD已经完全一致

22:让工作区的文件恢复成和暂存区一样

用git diff 文件名 查看，没有差别了

23:取消暂存区部分文件的更改

用status查看，没有了这个变更
文件名处可以用空格添加多个文件

24:消除最近的几次提交

回退到以前的状况

暂存区和工作区也回到原来状态，后续的变更都会丢失

25:看看两次commit之间的文件差异

26:正确删除文件

在工作区删除

在暂存区把工作文件删除

恢复文件

直接用git rm可以直接删除

27:开发中临时加塞的紧急任务

存放到下一步不影响工作区的环境中
再使用git status可以发现现在的工作区是干净的，可以做紧急任务

继续做变更

工作区的文件就会恢复到之前的状况
作用：1.将之前存放的内容弹出到工作区 2.stash列表中还在 可以反复使用

使用pop，会丢掉stash里面的信息

28:指定不需要git管理的文件

.gitignore
*.文件类型

doc文件夹下的文件git不管理
doc/ 也可以 不管理下面的文件但会管理doc文件夹

29:将git仓库备份到本地

传输协议：本地协议（哑协议/智能协议）、http/https协议（用户名和密码）、ssh协议（公私钥）
哑协议传输进度不可见；智能协议传输进度可见
智能协议传输速度比哑协议快

备份特点：可以多点备份
哑协议

拷贝pwd展示的地址

智能协议

有进度条

将本地信息更新到远端

分支push到了远端
