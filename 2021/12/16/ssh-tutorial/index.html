A simple guide for connecting to shark machines

This is a simple guide for you to use VS Code to connect to a shark machine and help you get started smoothly.

Disclaimer:

The course staff generally don’t help with VS Code issues but we will try our best.

The following instructions are only one of the ways to connect to shark machines, you are free to use any methods. There is no preference.

First, what is a shark machine?

Please refer to the page: http://www.cs.cmu.edu/~213/labmachines.html to get a general idea of the shark machines.

Third, how to use VS Code to help?

Remote extension. The official guidance can be found on this website: https://code.visualstudio.com/docs/remote/ssh

S1: download the extension from the extension marketplace

S2: set up the ssh connection.

S3: connect to the shark machine (you can choose anyone from the list on the course website)

choose Linux when connecting! and then input your passwords.

S4: go back to the explorer, and open the folder. You may need to re-enter your passwords;

(chances are that there is nothing in the /private folder)

Then you are all set. You can edit remote files in VS Code. You can drag files from local to the remote folder also.

Fourth, if you’d prefer CLion on Mac?

Official guideline can be found here: https://www.jetbrains.com/help/clion/create-ssh-configurations.html

S1: After creating an empty directory, in the Settings/Preferences dialog ⌘, (command+comma), search for deployment.

S2: in deployment, click “+” and choose “SFTP”. For the “New server name”, type your favorite Shark machine, i.e. angelshark.ics.cs.cmu.edu.

S3: click the “…” to add a configuration of SSH.

S4: in the SSH Configurations window, follow these steps. The user name should be your andrewid, and the password is the one you use for ssh to Shark. After you “Test Connection”, if it pops a window saying “successfully connected”, you are good! Then, click “Apply” and “OK”.

S5: Then, you are back to the Preferences window. You could click “Autodetect” to search for your working directory in Shark. Usually, if the local directory you create has the same name as the remote directory on Shark, it will be autodetected and the path is the full path. Otherwise, you need to put the full path manually. To find your pull path of your remote directory of your lab, you could type “pwd” in the remote directory and copy & paste it here.

Note: if it says “Connection failed”, sometimes you need to “Test Connection” again or do these steps with connecting to CMU VPN.

S6: Among the navigation bars, in “Tools”, you can see “Browse Remote Host” under “Deployment”. Click it, and you will see a list of files in your remote directory of your lab. You could click on any file you want to edit.

S7: After you make any changes, you will see a header saying “This file has been modified. Upload?”. There are two options: one is to revert to the last uploaded version, the other is to upload current version to remote. This allows you to edit files on CLion without scp from/to remote to local over and over again.

S8: Enjoy!

my AFS is full how do I clear it up?

If you use VSCode, try deleting the directory that has all of your connections with VSCode. It can get very bloated, very quickly. I had this issue on Wednesday and after deleting my VSCode directory it went from 100% storage to 4%.

The command is rm -rf .vscode-server, from your root directory.

Cannot Access to Shark Machine with VSCode

I cannot access to the shark machine with my VSCode. It just jumping out the password box and ask me to input the password again and again without really connecting to it. I wonder how could I solve this? Thank you

Not sure if this will solve your issue but this is a fix I’ve used for repetitive password prompting:

  1. Open the Command Palette (control-shift-p)
  2. Search for “Kill VS Code Server on Host…” and select it
  3. Select the shark server you’re trying to connect to (shark.andrew.cmu.edu or whichever specific shark you’re using)
  4. Go to VS Code settings (Control-comma)
  5. Type remote.ssh.useFlock into the search field
  6. Change the “Remote.SSH: Use Flock” checkbox (i.e., check it if it’s unchecked, or uncheck it if it’s checked)
  7. Restart VS Code and try logging in over SSH.
